# 部署與重現指南

更新日期：2026-05-26

這份文件的目的只有一個：讓組員可以最短時間重現目前可測的功能，而不是一次把所有規劃功能都裝起來。

## 路線選擇

### 路線 A：純筆電快速測試

適合想先確認 LLM API Server、狀態頁與聊天頁是否可用的人。

### 路線 B：筆電 + RPi4 完整流程

適合要驗證人臉辨識、VIP 註冊與區網串接的人。

## 前置條件

### 筆電端

- Windows 10/11
- 已安裝 Ollama
- 建議有 NVIDIA GPU，但不是必要條件
- Python 3.10+

### RPi4 端

- Raspberry Pi 4，建議 8GB RAM
- Ubuntu 24.04 aarch64
- 攝影機可用
- 可連到與筆電同一個區網

## 路線 A：純筆電快速測試

### 1. 下載模型並建立預設模型

```bash
ollama pull qwen2.5:3b
ollama create qwen-bank -f Modelfile_qwen
```

如果要測比較模型：

```bash
ollama pull gemma3:4b
ollama create gemma3-bank -f Modelfile_gemma3
ollama pull gemma4:e2b
ollama create gemma4e2b-bank -f Modelfile_gemma4e2b
```

### 2. 安裝 Python 依賴並啟動 API Server

```bash
python -m pip install fastapi uvicorn pydantic
python src/llm_server/main.py --host 0.0.0.0 --port 8000
```

### 3. 驗證服務

瀏覽器開啟：

- http://localhost:8000/
- http://localhost:8000/chat
- http://localhost:8000/health

或直接用命令測：

```bash
curl http://localhost:8000/health
```

### 4. 最小 smoke test

在聊天頁測三種情境：

1. VIP
2. blacklist
3. visitor

預期結果：

- 有正常 JSON 結果
- 狀態頁可看到請求紀錄
- 若模型關閉，仍能看到 fallback 回應

## 路線 B：筆電 + RPi4 完整流程

### 1. 筆電端先啟動 API Server

照路線 A 完成即可。

若要讓區網裝置可呼叫 Ollama，請另外確認 Ollama 的 listen 設定符合你的環境；若只需要讓 RPi4 呼叫 FastAPI，至少要保證 port 8000 可被區網存取。

### 2. 在 RPi4 下載專案與部署環境

```bash
git clone https://github.com/xiange1209/welcoming_robot.git ai_bank_robot
cd ai_bank_robot
bash setup_ubuntu.sh
source ~/ai_bank_robot_env/bin/activate
```

如果舊文件或舊腳本還提到 setup_rpi4_quick.sh，可以直接忽略；它現在只是 setup_ubuntu.sh 的相容入口。

### 3. 初始化資料庫並新增測試 VIP

```bash
python3 src/scripts/manage_vip_database.py init
python3 src/scripts/manage_vip_database.py add-vip-image "測試VIP" photo.jpg --gender M --level gold
python3 src/scripts/manage_vip_database.py list-vips
```

人臉註冊有兩種方式：

- 單張照片：`add-vip-image` / `add-blacklist-image`
- 攝影機拍攝：`add-vip-camera` / `add-blacklist-camera`

若你要驗證單樣本辨識，先用單張照片註冊；若你要提高穩定度，再改用攝影機拍攝多張樣本註冊。

### 4. 驗證與筆電的區網連線

在 RPi4 執行：

```bash
curl http://<筆電IP>:8000/health
```

回傳 online 才代表 FastAPI 連得到。這一步通過後，才值得做後續的 /api/chat 串接測試。

### 5. 啟動即時人臉辨識

```bash
export DISPLAY=:1
python3 src/scripts/realtime_detection_insightface.py
```

### 5.5 先做單張照片辨識驗證

在進入即時辨識前，建議先用單張照片驗證資料庫與閾值是否合理：

```bash
python3 src/scripts/test_vip_recognition.py path/to/vip.jpg --expect-type vip --expect-name "測試VIP"
python3 src/scripts/test_vip_recognition.py path/to/blacklist.jpg --expect-type blacklist --expect-name "測試黑名單"
python3 src/scripts/test_vip_recognition.py path/to/visitor.jpg --expect-type visitor
```

這個腳本會列出：

- 最終辨識結果
- 信心度
- VIP / 黑名單前幾名候選

如果這一步就已經不準，先不要急著做即時 GUI 驗證，先回頭調整照片品質、資料庫與閾值。

### 6. 完整 smoke test 清單

請依序確認：

1. list-vips 看得到剛註冊的 VIP。
2. 即時辨識視窗能正常開啟。
3. 臉部出現在鏡頭前時有框線與標籤。
4. 已註冊 VIP 能在合理角度與光線下被辨識。
5. 從 RPi4 可以持續呼叫筆電 /health。

## 建議的測試記錄格式

每位組員完成測試後，至少留下：

- 測試日期
- 硬體環境
- 使用模型
- 是否成功啟動 /health、/chat、即時辨識
- 是否成功完成 VIP 註冊與辨識
- 失敗點與錯誤訊息

## 目前明確尚未驗證的地方

- RPi4 透過程式自動呼叫 /api/chat 的實機紀錄
- 活體檢測 ARM64 穩定性
- Gemma3 / Gemma4 在筆電上的延遲與 JSON 成功率
- ROS 2 bringup 與導航

## 若只剩很少時間，要先測什麼

1. 先跑路線 A，確保 API Server 與聊天頁正常。
2. 再跑路線 B 的 init、add-vip-image、realtime_detection_insightface。
3. 最後才測模型切換與 Gemma 比較。
