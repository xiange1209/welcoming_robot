# 部署與重現指南

更新日期：2026-07-02

這份文件的目的只有一個：讓組員可以最短時間重現目前可測的功能，而不是一次把所有規劃功能都裝起來。

> **重要**：2026-07-02 專案已重組為 ROS 2 workspace（以同學的 smartnav-bot 為主體）。
> 舊的 FastAPI llm_server、`src/scripts/`（manage_vip_database.py、realtime_detection_insightface.py 等）、
> `setup_ubuntu.sh`、`setup_rpi4_quick.sh` 已全部刪除。凡是還提到這些路徑或指令的舊文件一律作廢，以本文件為準。

## 路線選擇

### 路線 A：純筆電快速測試

適合想先驗證 Ollama 模型延遲、確認哪個模型能塞進 4GB VRAM 的人。不需要 RPi4、不需要 ROS 2。

### 路線 B：RPi4 + 筆電完整 ROS 2 流程

適合要驗證人臉辨識、人臉註冊、辨識結果→文字橋接→LLM 串接的人。這條路線目前 **尚未在 RPi4 實機跑過一次完整流程**，第一個照做的人請把每一步的成功/失敗記下來。

## 前置條件

### 筆電端

- Windows 10/11
- Ollama（實測版本 0.24.0）
- Python 3.10+（Windows 上指令是 `python`，不是 `python3`）
- 建議有 NVIDIA GPU；實測環境為 RTX 3050 Laptop 4GB VRAM，本文件的延遲數據都以此為基準

### RPi4 端

- Raspberry Pi 4，建議 8GB RAM
- Ubuntu 24.04 aarch64
- ROS 2 Jazzy（安裝步驟見路線 B）
- 攝影機可用（image topic 預設 `/image_raw`）
- 麥克風/喇叭（只有測 audio 才需要）
- 與筆電在同一個區網

## 路線 A：純筆電快速測試

### 1. 安裝 Ollama 並下載基底模型

先到 https://ollama.com/download 安裝 Windows 版 Ollama，然後：

```bash
ollama pull qwen2.5:3b
ollama pull gemma3:4b
```

如果磁碟空間和時間允許，再抓 gemma4 系列做完整比較（檔案很大：e2b 約 7.2GB、e4b 約 9.6GB，而且實測在 4GB VRAM 上會非常慢）：

```bash
ollama pull gemma4:e2b
ollama pull gemma4:e4b
```

### 2. 建立 bank 模型

在專案根目錄（有 Modelfile 的那層）執行：

```bash
ollama create gemma3-bank -f Modelfile_gemma3
ollama create gemma4e2b-bank -f Modelfile_gemma4e2b
ollama create gemma4e4b-bank -f Modelfile_gemma4e4b
ollama create qwen-bank -f Modelfile_qwen
```

qwen2.5:3b 也可以不建 bank 版直接測，2026-07-02 的實測表就是直接用 `qwen2.5:3b` 原始 tag。

### 3. 跑 LLM 延遲基準測試

```bash
python scripts\benchmark_llm_models.py --models gemma3-bank gemma4e2b-bank gemma4e4b-bank qwen2.5:3b
```

注意：

- Windows 上用 `python`，不是 `python3`。
- **一定要自己指定 `--models`**，腳本的預設模型清單含已不存在的 `gemma4-bank`，直接跑預設會有一項失敗。
- 每個模型會先做 1 次暖機呼叫（不計入統計），再跑 5 次正式量測。
- 結果會印出可直接貼進報告的 Markdown 表，並輸出 JSON 明細（預設 `benchmark_llm_results.json`）。

### 4. 預期結果（2026-07-02 實測，RTX 3050 Laptop 4GB / Ollama 0.24.0）

| 模型 (Ollama tag) | 基底 | 首次輸出平均 (ms) | 總生成平均 (ms) | 最快 (ms) | 最慢 (ms) | 成功率 |
|---|---|---|---|---|---|---|
| gemma3-bank | gemma3:4b | 2303 | 4700 | 4583 | 4755 | 100% |
| gemma4e2b-bank | gemma4:e2b | 42235 | 47909 | 44838 | 50206 | 100% |
| gemma4e4b-bank | gemma4:e4b | 95066 | 107123 | 100231 | 123839 | 100% |
| qwen2.5:3b | qwen2.5:3b | 2218 | 3076 | 3017 | 3196 | 100% |

關鍵解讀：

- gemma4 系列模型檔（7.2GB / 9.6GB）超出 4GB VRAM，Ollama 會卸載到 CPU，導致慢 20~40 倍，在這台筆電上不適合即時互動。
- gemma3:4b（約 4GB）與 qwen2.5:3b（約 1.9GB）塞得進 VRAM；**qwen2.5:3b 最快，gemma3:4b 次之**。
- 你跑出來的絕對數字會因機器不同而異，但「gemma4 慢一個數量級」這個相對關係在同級 GPU 上應該重現得出來。若差距消失，先確認你的 GPU VRAM 大小。

### 5.（可選）直接測對話

```bash
ollama run qwen2.5:3b
```

隨便打一句「你好，我想辦理黃金存款」，確認模型會正常回覆即可。

## 路線 B：RPi4 + 筆電完整 ROS 2 流程

### 1. RPi4 安裝 Ubuntu 24.04 + ROS 2 Jazzy

Ubuntu 24.04（aarch64）裝好後，照 ROS 2 官方步驟安裝 Jazzy：

```bash
# locale
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8

# 加入 ROS 2 apt 來源（官方 ros2-apt-source 套件）
sudo apt install software-properties-common curl -y
sudo add-apt-repository universe
export ROS_APT_SOURCE_VERSION=$(curl -s https://api.github.com/repos/ros-infrastructure/ros-apt-source/releases/latest | grep -F "tag_name" | awk -F\" '{print $4}')
curl -L -o /tmp/ros2-apt-source.deb "https://github.com/ros-infrastructure/ros-apt-source/releases/download/${ROS_APT_SOURCE_VERSION}/ros2-apt-source_${ROS_APT_SOURCE_VERSION}.$(. /etc/os-release && echo $VERSION_CODENAME)_all.deb"
sudo apt install /tmp/ros2-apt-source.deb

# 安裝 ROS 2 Jazzy 與開發工具
sudo apt update && sudo apt upgrade -y
sudo apt install ros-jazzy-ros-base ros-dev-tools -y   # RPi4 建議 ros-base；要 RViz 等 GUI 才裝 ros-jazzy-desktop

# 每次開 shell 都要 source，建議寫進 ~/.bashrc
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source /opt/ros/jazzy/setup.bash
```

### 2. Clone 專案

```bash
cd ~
git clone https://github.com/xiange1209/welcoming_robot.git smartnav_ws
cd ~/smartnav_ws
```

repo 根目錄就是 ROS 2 workspace 根目錄（裡面已有 `src/`），不需要再包一層。

### 3. Clone frontier_exploration_ros2

同學的 repo 用 git submodule 帶這個套件，本 repo 尚未加 submodule，所以要自己 clone 到 `src/`：

```bash
git clone https://github.com/mertgulerx/frontier_exploration_ros2 src/frontier_exploration_ros2
```

少了這步，`smartnav_navigation` 的 brain.launch.py（會啟動 frontier_explorer）會起不來。

### 4. rosdep 安裝系統依賴

```bash
sudo rosdep init   # 只有第一次需要；已 init 過會報錯，忽略即可
rosdep update
rosdep install --from-paths src --ignore-src -r -y
```

### 5. pip 安裝各套件 Python 依賴

```bash
pip install -r src/smartnav_vision/requirements.txt
pip install -r src/smartnav_audio/requirements.txt
pip install -r src/smartnav_llm/requirements.txt
```

Ubuntu 24.04 的系統 Python 有 PEP 668 保護，若出現 `externally-managed-environment` 錯誤，在每行後面加 `--break-system-packages`（ROS 2 節點跑在系統 Python 上，用獨立 venv 反而會找不到套件，除非 venv 建立時加 `--system-site-packages`）。

InsightFace buffalo_sc 模型會在 vision 節點第一次啟動時自動下載，請確保當下有網路。

### 6. 下載語音模型

```bash
bash scripts/download_audio_models.sh
```

在專案根目錄執行。會下載 sherpa-onnx 的 KWS/ASR/VAD/TTS 模型到 `src/smartnav_audio/models/`。只測 vision + LLM 的人可以跳過這步，啟動時記得 `enable_audio:=false`。

### 7. 建置 workspace

```bash
cd ~/smartnav_ws
colcon build --symlink-install
source install/setup.bash
```

**這是全新結構第一次在 RPi4 上建置，預期可能踩雷。** 若個別套件建置失敗，先用 `--packages-select` 逐一定位：

```bash
colcon build --symlink-install --packages-select smartnav_msgs
colcon build --symlink-install --packages-select smartnav_vision
```

`smartnav_msgs` 一定要最先建成功，其他套件都依賴它。

### 8. 筆電 Ollama 對區網開放，RPi4 驗證連線

筆電端（Windows）讓 Ollama 監聽所有介面：

```powershell
setx OLLAMA_HOST 0.0.0.0
```

設完後**完全結束並重啟 Ollama**（系統匣圖示 Quit 再開），並確認 Windows 防火牆放行 11434 埠。

RPi4 端驗證：

```bash
curl http://<筆電IP>:11434/api/tags
```

有回傳 JSON 模型清單才算通。這步不通，後面 LLM 串接全部免談，先解決網路再繼續。

### 9. 啟動系統

完整啟動（vision + audio + llm + navigation）：

```bash
ros2 launch smartnav_bringup smartnav.launch.py \
  ollama_base_url:=http://<筆電IP>:11434 \
  model_name:=qwen2.5:3b \
  enable_gpu:=false
```

說明：

- `model_name` 預設是 `gemma4:e2b`，但依 2026-07-02 實測，在 RTX 3050 4GB 上請改用 `qwen2.5:3b`（最快）或 `gemma3-bank`。
- `enable_gpu` 預設 `true`，RPi4 沒有可用 GPU，請設 `false`。
- 硬體不齊時用開關逐步啟動，例如只測 vision + LLM：

```bash
ros2 launch smartnav_bringup smartnav.launch.py \
  ollama_base_url:=http://<筆電IP>:11434 \
  model_name:=qwen2.5:3b \
  enable_gpu:=false \
  enable_audio:=false \
  enable_navigation:=false
```

- 其他可用參數：`image_topic`（預設 `/image_raw`）、`use_sim_time`、`enable_vision`、`enable_llm`、`enable_nav2`（預設 `false`，需要 TurtleBot3 本體或模擬環境才開）。

### 10. 人臉註冊與辨識驗證

啟動後，先確認節點與話題都在：

```bash
ros2 node list          # 應看到 face_recognition_node、face_registration_node、recognition_text_bridge_node 等
ros2 topic list         # 應看到 /face_recognition/result、/user_text
```

**註冊 VIP**（本人站到鏡頭前再呼叫，節點會從相機連續取樣）：

```bash
ros2 service call /face_registration/register smartnav_msgs/srv/RegisterFace \
  "{person_name: '陳佳憲', gender: 'M', person_type: 'VIP', num_samples: 10}"
```

**註冊黑名單**：

```bash
ros2 service call /face_registration/register smartnav_msgs/srv/RegisterFace \
  "{person_name: '測試黑名單', gender: 'F', person_type: 'BLACKLIST', num_samples: 10}"
```

回應會帶 `success`、`message` 與分配的 `person_uuid`。註冊節點成功後會自動通知辨識節點刷新快取；若辨識結果遲遲沒更新，可手動刷新：

```bash
ros2 service call /face_recognition/refresh_cache std_srvs/srv/Empty
```

**驗證辨識結果**：

```bash
ros2 topic echo /face_recognition/result
```

站到鏡頭前，應看到 `person_name`、`gender`、`person_type`（VIP/BLACKLIST/VISITOR）、`confidence` 等欄位。注意：同一個人持續在鏡頭前時，結果重發間隔預設 15 秒（`result_republish_interval`），不是每一幀都發，不要誤以為當機。

**驗證辨識→文字橋接**（這是進 LLM 的入口）：

```bash
ros2 topic echo /user_text
```

應看到橋接節點把辨識結果轉成的自然語言句子。若要看標框畫面，把 vision 節點的 `publish_debug_image` 參數設為 true，訂閱 `/face_recognition/debug_image`（VIP=綠框、黑名單=紅框、訪客=藍框）。

## 完整 smoke test 清單

照新架構逐項確認，能打幾個勾就代表重現到哪：

1. 筆電 `ollama list` 看得到 qwen2.5:3b 與各 bank 模型。
2. 筆電 `python scripts\benchmark_llm_models.py --models qwen2.5:3b` 能跑完並輸出表格。
3. RPi4 `colcon build --symlink-install` 全部套件建置成功、無失敗。
4. RPi4 `curl http://<筆電IP>:11434/api/tags` 有回傳。
5. `ros2 launch smartnav_bringup smartnav.launch.py ...` 啟動無 crash（先用 `enable_audio:=false enable_navigation:=false` 縮小範圍）。
6. `ros2 topic echo /face_recognition/result` 在有人臉時有輸出。
7. `/face_registration/register` 註冊成功並回傳 person_uuid。
8. 註冊過的人再次入鏡，`person_type` 正確顯示 VIP 或 BLACKLIST，且 `gender` 正確。
9. `ros2 topic echo /user_text` 看得到橋接後的自然語言文字。
10. smartnav_llm 節點 log 顯示有收到 /user_text 並向筆電 Ollama 發出請求、拿到回覆。
11. （有下載語音模型且有麥克風/喇叭時）audio 四個節點啟動無錯誤。
12. （有 TurtleBot3 或模擬環境時）`enable_nav2:=true` 能起 Nav2 棧。

## 目前明確尚未驗證的地方

誠實列出，避免報告寫過頭：

- **整個 ROS 2 workspace 尚未在 RPi4 實機 colcon build 過**，也沒有實機運行紀錄。路線 B 的第 4~10 步全部屬於「已實作、待實機驗證」。
- vision → recognition_text_bridge → /user_text → smartnav_llm 的端到端串接未實測。
- smartnav_audio（sherpa-onnx 喚醒/ASR/TTS/播放）未在 RPi4 實測。
- smartnav_navigation 與 Nav2 未實測；frontier_exploration_ros2 需另行 clone 也尚未在本 repo 驗證能一起建置。
- **smartnav_llm 的工具集全是導航導向**（create_map / navigate / waypoint 等），沒有銀行場景工具（帶 VIP 到貴賓室、通報行員），system_prompt.txt 也是導航機器人設定。人臉事件進 /user_text 後只會被當一般對話處理——這是已知的功能缺口，不是 bug。
- 已驗證的只有兩件事：(1) RPi4 上 InsightFace buffalo_sc 即時人臉檢測 15+ FPS（在舊架構驗證，辨識引擎與 ROS 2 版相同）；(2) 筆電上四個模型的 LLM 延遲基準（上表）。

## 若只剩很少時間，要先測什麼

1. 先跑路線 A：pull qwen2.5:3b、跑一次 benchmark，確認筆電 LLM 可用且延遲合理。
2. 再跑路線 B 的第 1~7 步，目標只有一個：**在 RPi4 上把 workspace 建起來**。這是目前最大的未知數。
3. 建置成功後，用 `enable_audio:=false enable_navigation:=false` 只起 vision + LLM，走完註冊→辨識→/user_text 這條線。
4. audio 與 navigation 留到最後，缺硬體或缺時間就直接關掉對應開關。
