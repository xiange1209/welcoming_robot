## 🛠️ 技術堆棧確認

### 核心推理引擎
- **人臉檢測**: InsightFace buffalo_sc (ONNX, INT8)
- **人臉識別**: 512D embedding cosine similarity + k-NN voting
- **數據增強**: InsightFace 3D rotation synthesis
- **活體檢測**: MediaPipe Face Landmarks (if available on RPi4)

### 語言模型與語音
- **STT**: OpenAI Whisper tiny (INT4 quantized)
- **LLM**: Qwen2.5-1.5B (INT4 quantized, ~1GB, ~1-2s/response)
- **LLM 增強**: RAG (FAISS/ChromaDB) + 銀行業務知識庫
- **TTS**: pyttsx3 (本地文本轉語音, 中英文)

### 數據庫與存儲
- **本地**: SQLite (vip_members, vip_embeddings, blacklist, visit_logs, security_alerts)
- **同步**: PC Web UI → SCP → RPi4

### ROS 2 架構
- **框架**: ROS 2 Jazzy
- **節點**: camera_node, face_recognition_node, brain_node, voice_node, navigation_node
- **通訊**: Topic (`/camera/info`, `/face_recognition/detections`), Service (registration)
- **機器人**: TurtleBot3 (Cmd_Vel 控制, 自動導航)

### 開發工具
- **主菜單**: `src/scripts/main.py` (交互式 CLI)
- **資料庫管理**: `src/scripts/manage_vip_database.py`
- **測試腳本**: `src/scripts/benchmark.py`, `src/scripts/test_vip_recognition.py`
- **實時檢測**: `src/scripts/realtime_detection_insightface.py`

## 🚨 已知風險與應對

| 風險 | 影響 | 應對方案 |
|------|------|---------|
| **MediaPipe ARM64** | 活體檢測無法部署 | 先使用簡化邏輯 (眨眼檢測), Phase B+ 考慮替代方案 (dlib) |
| **Qwen INT4 量化 > 1GB** | RPi4 內存爆炸 | 試 Qwen1.5B 或 Gemma-2B, 考慮 PC fallback via ROS 2 |
| **Whisper tiny 推理 > 5s** | 對話延遲過長 | 預錄常見問題回覆, 或 PC 端 Whisper fallback |
| **SLAM 地圖建立失敗** | TurtleBot3 定位偏差 | 使用預製地圖 or 簡化為固定路線 |
| **多客戶同時檢測** | 人臉檢測吞吐不足 | frame skipping (每 2-3 幀檢測一次) + 隊列管理 |
