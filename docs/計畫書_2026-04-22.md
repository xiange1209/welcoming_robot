## 🎯 當前困難 & 解決方案

### 困難 1: 跨設備識別準確率低

**現象**: 
- iPhone 照片: 相似度 0.78 ✅
- 監控攝像機: 相似度 0.38 ❌

**根本原因**:
```
不同相機 → ISP 後處理差異 → 特徵漂移
```

**解決方案**:
```
Step 1: 色彩空間轉換 (RGB → LAB)
  └─ 補償光線差異

Step 2: 直方圖均衡化
  └─ 補償曝光差異

Step 3: 多尺度特徵融合
  └─ 3 個不同尺度的 embedding

Step 4: 特徵歸一化
  └─ |embedding| = 1 (單位向量)

預期效果: 相似度 0.38 → 0.65
識別率提升: 45% → 70% (+25%)
```

**實現時間**: 2-3 天

---

### 困難 2: 單樣本識別準確率低

**現象**:
- 單樣本 (1 張): 40% 識別率 ❌
- 多樣本 (10 張): 92% 識別率 ✅

**根本原因**:
```
姿態、光線、表情變化 → embedding 不穩定
```

**解決方案**:
```
Step 1: 特徵擴增
  ├─ 水平翻轉
  ├─ 亮度增強
  ├─ 亮度減弱
  └─ 高斯模糊 (共 5 個)

Step 2: 統計聚類中心
  ├─ 計算平均值
  ├─ 移除異常值 (MAD)
  └─ 重新計算 (robust mean)

Step 3: 動態閾值
  ├─ 單樣本模式: 0.60
  └─ 多樣本模式: 0.65

預期效果: 40% → 75% 識別率
```

**實現時間**: 3-4 天

---

### 困難 3: 活體檢測無法安裝

**現象**:
```
$ pip install mediapipe
ERROR: No matching distribution for mediapipe (ARM64)
```

**根本原因**: MediaPipe 官方未提供 ARM64 輪子

**解決方案**:
```
方案 A: Dlib 眨眼檢測 (優先)
  ├─ 安裝: pip install dlib
  ├─ 輕量級 (50MB)
  ├─ 快速推理 (~30ms)
  └─ 準確度: 75%

方案 B: mediapipe-lite (備選)
  └─ 非官方包，實驗性

預期效果: 活體檢測 75% 準確度
```

**實現時間**: 1-2 天
