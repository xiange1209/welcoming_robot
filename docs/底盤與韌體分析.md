# 底盤與韌體分析（STM32 × ROS2 整合原理）

> 2026-07-09 由三個平行 agent 深讀產出：`F103RCT6_阿克曼_移動機器人STM32源碼`（Keil 工程）、
> `src/wheeltec/turn_on_wheeltec_robot`（已整合的 Jazzy 修正版驅動）、akm 導航參數與廠商說明文件。
> 所有數據皆有源碼檔名行號可回溯；不確定處誠實標註於文末。

## 一、三層架構：誰負責什麼

```
┌─────────────────────────────────────────────────────────┐
│ 第 3 層：智慧層（我們寫的，RPi4）                          │
│ smartnav_vision/audio/llm → nav2 goal → Nav2(MPPI/Smac)  │
└──────────────────────────┬──────────────────────────────┘
                           │ /cmd_vel (geometry_msgs/Twist)
┌──────────────────────────┴──────────────────────────────┐
│ 第 2 層：ROS2 驅動層（廠商程式碼，已 Jazzy 化，RPi4）       │
│ turn_on_wheeltec_robot：cmd_vel→串口幀；串口幀→odom+IMU    │
│ robot_localization EKF → odom_combined TF                │
└──────────────────────────┬──────────────────────────────┘
                           │ USB 串口 /dev/wheeltec_controller
                           │ 115200 8N1，幀頭 0x7B 幀尾 0x7D，XOR 校驗
┌──────────────────────────┴──────────────────────────────┐
│ 第 1 層：STM32F103RCT6 韌體（廠商預設，不用改）             │
│ FreeRTOS 6 任務；100Hz 控制迴圈；阿克曼逆運動學；            │
│ 增量式 PI 輪速閉環；舵機轉向；MPU6050；電壓監控；安全看門狗   │
└─────────────────────────────────────────────────────────┘
```

**分工原則**：STM32 管「怎麼動」（運動學、PID、PWM），ROS2 管「往哪動」（里程計、定位、路徑），
智慧層管「為什麼動」（任務決策）。兩邊只透過一條 115200 串口與兩種資料幀溝通。

## 二、串口協議（兩邊源碼互相印證，欄位完全對上）

**下行（RPi4→STM32，11 bytes，隨 /cmd_vel 觸發）**：
`[0]0x7B｜[1]回充旗標｜[2]安全旗標｜[3:4]Vx｜[5:6]Vy｜[7:8]Vz（short 大端，m/s×1000）｜[9]XOR(0-8)｜[10]0x7D`

**上行（STM32→RPi4，24 bytes @20Hz）**：
`[0]0x7B｜[1]Flag_Stop｜[2:7]實測 Vx/Vy/Vz｜[8:19]IMU 六軸原始值｜[20:21]電壓 mV｜[22]XOR(0-21)｜[23]0x7D`

- 韌體收到 (Vx,Vz) 後自行換算前輪轉角：`AngleR=atan(軸距/(Vx/Vz+0.5×輪距))`，**並強制最小轉彎半徑**
  （高配 0.75m／頂級擺懸 1.4m／頂級獨立懸 1.2m）——超過物理極限的指令會被韌體截斷，上層不用處理。
- ROS 端 odom 由上行速度積分（`wheeltec_robot.cpp:884-886`），IMU 走互補濾波出 `imu/data_raw`，
  兩者由 EKF 融合成 `odom_combined`（frame 與 TF 的唯一來源是 EKF，驅動節點本身不發 TF）。
- 上行另有超音波幀（0xFA/0xFC, 19B）與自動回充幀（0x7C/0x7F, 8B），同一串口分幀頭並行解析。

## 三、韌體端要知道的行為（除錯必備）

| 行為 | 細節 | 對我們的影響 |
|---|---|---|
| 命令看門狗 | 1 秒沒收到串口命令→目標速度清零 | ROS 節點掛掉車會自己停，安全 |
| 開機保護 | 上電 10 秒內不理會串口命令 | 開機馬上發 cmd_vel 沒反應是正常 |
| 控制優先序 | 藍牙 APP＞航模遙控＞PS2＞串口(ROS) | **APP 連著時 ROS 指令無效**，測試前先斷 APP |
| 低壓保護 | 電池 <20V 直接關 PWM | 沒電時「不動」不是程式 bug |
| 車型自檢 | 開機自檢輸出連續近滿幅→判定車型選錯→鎖車 | 板上電位器選錯 Car_Mode 會鎖 |
| 舵機零點 | 存在 STM32 內，**刷韌體會清掉** | 刷頂級阿克曼韌體前必用藍牙 APP 讀出三個零點值（見韌體必讀 PDF） |

## 四、car_mode 對應（韌體 Car_Mode × ROS car_mode）

車子是**阿克曼**構型（前輪轉向、後輪差速驅動，同汽車）。韌體由板上電位器選型，ROS 由
`src/wheeltec/turn_on_wheeltec_robot/config/wheeltec_param.yaml` 的 `car_mode` 選型，**兩邊必須一致**：

| 廠商稱呼 | 韌體 Car_Mode | ROS car_mode | 最小轉彎半徑 | 特徵 |
|---|---|---|---|---|
| 高配阿克曼 | 0/1 | `senior_akm` | 0.80m | 胎徑 0.125m，開環舵機 |
| 頂級擺懸阿克曼 | 2/3 | `top_akm_bs` | 1.65m | 胎徑 0.18m，滑軌電位器閉環 |
| 頂級獨立懸（卡丁） | 4/5 | `top_akm_dl` | 1.78m | 胎徑 0.254m，滑軌電位器閉環 |
| 迷你阿克曼 | — | `mini_akm` | 0.35m | 小車 |

ROS 端 `car_mode` 影響四處：URDF 選檔、`robot_model.yaml` 靜態 TF、驅動內避障分支、
**Nav2 自動載入 `param_{car_mode}.yaml`**。

## 五、阿克曼導航與差速車的差異（Nav2 參數層）

- **Controller**：MPPI `motion_model: "Ackermann"`＋`AckermannConstraints.min_turning_r`；
  不用 RotationShim（**阿克曼不能原地旋轉**）；`enforce_path_inversion: true`（允許倒車換向）。
- **Planner**：SmacPlannerHybrid `motion_model_for_search: "REEDS_SHEPP"`（可倒車搜尋；差速用 DUBIN），
  `minimum_turning_radius` 放大到實車半徑。
- **behavior_server（spin/backup 恢復行為）廠商整段註解未啟用**。
- 銀行場景空間規劃要留意：senior_akm 迴轉需直徑 1.6m 以上的空間，走道太窄會規劃失敗。

## 六、廠商 Humble 源碼 vs 我們的 Jazzy 版

抽查 `turn_on_wheeltec_robot` 兩份為**同版基底**，專案側僅刻意修剪（移除 turtlesim/ackermann_msgs
死依賴與佔位回充服務、補 4 個 exec_depend、cmake 3.5→3.8）。`wheeltec_param.yaml` 兩邊完全相同。
**結論：不需要再碰廠商 Humble 原始包**——17 套件已移植並通過 x86（24/24）與 arm64（25/25）建置。

## 七、刷韌體注意（僅頂級阿克曼需要，一般不用刷）

出廠 hex 已在板上。若要刷 `F103RCT6_阿克曼小車STM32源碼_2023.04.10.hex`：
藍牙 APP（BT04-A，配對碼 1234）「獲取設備參數」→ 記下倒數第 2/3/4 行三個舵機零點 →
FlyMcu 燒錄（460800，勾校驗/編程後執行/RamIsp，勿勾連續燒錄）→ 等 10 秒紅燈閃 →
APP「發送所有參數」＋「設置掉電保存參數」。**沒記零點就刷會失去轉向標定。**
