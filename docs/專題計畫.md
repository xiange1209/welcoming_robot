## 一、專案目標

以 ROS 2 為主架構、WHEELTEC 阿克曼實車為載體的雙機系統，在銀行大廳場景完成：

1. **VIP 迎賓**：單張大頭照註冊，現場辨識 VIP／一般訪客／黑名單，辨識到 VIP 觸發語音迎賓與帶位。
2. **安全通報**：辨識到黑名單即時通報行員（推播＋現場警報畫面）。
3. **自然語言互動**：語音（喚醒詞＋ASR）→ LLM Agent 分析 → 呼叫 ROS2 工具（導航／銀行動作）→ TTS 回覆。
4. **人機介面**：平板網頁 HMI 顯示辨識結果、對話流、系統狀態、相機畫面。
5. **競賽目標（新增）**：下學期專題發表得名、參加校外專題競賽。評審視角的四個支柱——完整故事線（辨識→決策→行動→通報一氣呵成）、技術深度（邊緣-雲協作、交叉編譯部署）、量化驗證（八組實驗，見六）、誠實可信（三分驗證文化）。

## 二、系統架構

### 雙機協作

```
┌────────────────────────────────┐          ┌──────────────────────────┐
│ WHEELTEC senior_akm 實車        │          │ Windows 筆電              │
│ ├─ STM32 F103RCT6（底盤控制）   │  HTTP    │ (RTX 3050 4GB VRAM)      │
│ └─ RPi4B 4GB (Ubuntu 24.04,    │  Ollama  │ Ollama :11434            │
│    ROS2 Jazzy, ~/smartnav_ws)  ├─────────→│ (qwen2.5:3b 主力)         │
│    輕量節點：底盤/感測/HMI/橋接  │←─────────┤                          │
└────────────────┬───────────────┘  串流回覆 └──────────────────────────┘
                 │ rosbridge websocket :9090 / HTTP :8081 / MJPEG :8080
            平板（瀏覽器 HMI）
```

部署分工定調（2026-07-10）：**Pi 只跑輕量節點，重運算（vision／LLM Agent）移往筆電側**。分散式 ROS2 組態屬待驗證（筆電側需 Linux ROS2 環境，VMware bridged 或 WSL2）；已驗證退路是 Pi 跑全部節點＋筆電只供 Ollama HTTP（vision 在 Pi 有 15+ FPS 實測基礎）。

### 程式碼佈局（自包含 repo，2026-07-10 定案）

- `smartnav-bot/src/`——8 個自有套件：`smartnav_msgs`（介面）、`smartnav_vision`（InsightFace 人臉辨識）、`smartnav_audio`（sherpa-onnx 語音）、`smartnav_llm`（LangChain Agent＋`ollama_chat_bridge`）、`smartnav_navigation`（TB3/模擬導航）、`smartnav_brain`（空殼，路線圖 Phase B 填入）、`smartnav_hmi`（平板網頁 UI，已實作）、`smartnav_bringup`（統一啟動）。
- `src/`——17 個廠商套件（Humble 世代移植 Jazzy）：底盤 `turn_on_wheeltec_robot`、URDF、serial、雷達（ldlidar/lslidar）、astra 相機、麥克風陣列 `wheeltec_mic_ros2`、`wheeltec_robot_nav2`、slam 封裝、巡邏 `nav2_waypoint_cycle`、`ollama_ros_chat` 等。
- 事件流：相機 `/image_raw` → 人臉辨識 → `/face_recognition/result` → 轉自然語言 → `/user_text`（ASR 同入口）→ LLM Agent → `/llm_response`／`/speech_text` → TTS → 播放。

### 雙版本可選方案（bringup 參數切換，重疊功能各留兩版互為備援）

| 面向 | smartnav 版（預設） | wheeltec 版 |
|---|---|---|
| 語音 | sherpa-onnx 自由語句 | 麥克風陣列＋離線命令詞（`audio_stack:=wheeltec`） |
| LLM | LangChain Agent 含工具 | `ollama_ros_chat` 單輪對話（`llm_stack:=wheeltec`） |
| 導航 | `smartnav_navigation`（TB3/模擬，`odom` frame） | wheeltec 官方鏈（實車，`odom_combined` frame，不可混用） |

雙版本本身就是報告的比較實驗素材（見六、E5）。

## 七、風險與限制

1. **協作衝突風險**：同學（swient）保有本 repo push 權，且他的 repo 有未合併的 vision/brain 重設計；新功能現改由本側實作，`smartnav_brain` 動工前務必先對齊，避免兩邊各寫一版。
2. **astra 相機相容性**：閉源 .so 在 Ubuntu 24.04 未實測（B11 T3），失敗的退路是 usb_cam。
3. **分散式 ROS2 未驗證**：VMware NAT 下跨機 DDS 不通，需 bridged；失敗退路是已驗證的「Pi 跑全部＋筆電只供 Ollama」。
4. **LLM 模型限制**：4GB VRAM 上限鎖定 qwen2.5:3b／gemma3:4b；`model_name` 預設仍是 gemma4:e2b，啟動必須覆寫。
5. **阿克曼運動限制**：不能原地旋轉（迴轉半徑 0.80m），demo 動線與導航目標點要預留空間；兩條導航鏈 frame 不同（`odom_combined` vs `odom`）不可混用。
6. **通報服務選型**：LINE Notify 已終止服務；Telegram Bot API 為首選，需確認行員手機可安裝 Telegram（備選：Discord webhook、email）。
7. **wheeltec 離線語音是固定命令詞**：銀行詞彙需重編語法檔；自由對話走 smartnav 版。
