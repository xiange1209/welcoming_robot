## 🏗️ 系統架構

### 感測層 (Perception Layer)
```
相機驅動 (Picamera2/USB)
  ↓
麥克風驅動
  ↓
IMU / 里程計
```

### 處理層 (Processing Layer)
```
smartnav_vision (陳佳憲 + 孫瑋廷)
├─ face_recognition_node      (人臉識別)
└─ face_registration_node     (人臉註冊)

smartnav_audio (孫瑋廷)
├─ speech_recognizer_node     (語音識別框架)
└─ speech_synthesizer_node    (語音合成框架)

smartnav_brain (孫瑋廷)
└─ decision_engine_node       (決策引擎)
```

### 導航層 (Navigation Layer)
```
Nav2 (計畫規劃、路徑控制) - 隊友實現
```

### 執行層 (Execution Layer)
```
TurtleBot3 底盤驅動
```

## 🔴 當前困難與挑戰

### 困難 1: 跨設備人臉識別 ⚠️

**問題**: 用不同相機拍的照片無法相互識別
- iPhone 照片 (相似度 0.78) vs 監控攝像機 (0.38) ❌

**根本原因**: 相機 ISP 特性差異導致特徵漂移

**預期解決方案**:
- 色彩空間轉換 (RGB → LAB)
- 直方圖均衡化
- 多尺度特徵融合
- **預期效果**: 相似度提升 25%
- **工時**: 2-3 天

---

### 困難 2: 單樣本識別 ⚠️

**問題**: 銀行要求用一張照片識別，當前識別率低
- 單樣本 (1 張): 40% 識別率 ❌
- 多樣本 (10 張): 92% 識別率 ✅

**預期解決方案**:
- 特徵擴增 (翻轉、亮度調整)
- 統計聚類中心
- 動態閾值調整
- **預期效果**: 提升到 75% 識別率
- **工時**: 3-4 天

---

### 困難 3: 活體檢測 ⚠️

**問題**: MediaPipe 官方無 ARM64 支持

**預期解決方案**:
- 使用 Dlib 眨眼檢測 (輕量級替代方案)
- 或評估 mediapipe-lite 非官方包
- **預期效果**: 75% 準確度
- **工時**: 1-2 天
