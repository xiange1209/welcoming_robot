# 智慧銀行 VIP 迎賓與安全通報系統 - 項目規劃
## 基於樹莓派 4 的 AI 邊緣運算與語音辨識系統

---

## 項目概述

本專題為**樹莓派 4 + TurtleBot3** 版本的智慧銀行服務機器人系統，整合視覺 AI、語音交互、遠程通訊，在銀行環境中實現：

- 🔄 **視覺 AI**：
  - ✅ 人臉檢測（InsightFace RetinaFace，12-13 FPS）【Phase 2a 完成】
  - 🔄 人臉識別 + 活體防偽（VIP/黑名單比對，活體檢測）【Phase 2b 進行中】
  - ⏳ 表情檢測（EfficientNet-B0）【Phase 3 計劃】
- ✅ **語音交互**：Whisper STT（語音識別）+ pyttsx3（語音合成）+ 語義理解【Phase 3 計劃】
- ✅ **HMI 界面**：7 吋觸控屏實時信息展示【Phase 4 計劃】
- ✅ **LoRa 通訊**：VIP 通報推送 + 安全告警【Phase 4 計劃】
- ✅ **後端系統**：管理面板 + 數據同步 + MQTT 平台【Phase 5 計劃】
- ✅ **自主導航**：由同學負責 (ROS 2 + SLAM + Nav2)【Phase 3 計劃】

## 針對樹莓派 4 的優化策略

### 🎯 模型量化與輕量化

由於樹莓派 4 的 RAM (8GB) 和 CPU (4 核 ARM) 資源有限，所有 AI 模型必須高度優化：

#### 1. 深度學習推理框架
- ❌ **不用 TensorRT**（Jetson 專用）
- ✅ **使用 ONNX Runtime** (ARM 優化版本)
- ✅ **使用 TensorFlow Lite** (ARM 優化版本)
- **模型精度**：INT8 量化，保證 < 100MB 大小

#### 2. 人臉識別模型
- **RetinaFace** (量化版，< 10MB)
- **ArcFace** 輕量版 (< 20MB，生成 128D 向量)
- 優先使用 ONNX 模型而非 PyTorch

#### 3. 表情分類模型
- **MobileNetV2** 或 **EfficientNet-B0** (< 15MB)
- 使用預訓練的量化版本

#### 4. 性能指標

**Phase 2a 完成（人臉檢測）- 實際測試數據：**
- 人臉檢測：100-150ms / 幀（已驗證 12-13 FPS）
- 內存占用：350-400 MB
- CPU 使用率：60-70%（4 核 ARM，單核留裕度）
- 模型大小：~50 MB (buffalo_sc)

**Phase 2b 目標（人臉識別 + 活體檢測）：**
- 人臉識別：<300ms（512D embedding 與 VIP 資料庫比對）
- 活體檢測：<100ms（眨眼/搖頭/張嘴偵測）
- **整體推理延遲：< 500ms @ 10 FPS**

### ⚙️ 幀跳最佳化架構（Frame Skipping）

樹莓派 4 採用幀跳優化而非多線程（避免爭奪 CPU）：

```
Phase 2a (檢測) - 當前實現：
┌─────────────────────────────┐
│ 幀隊列（每幀 33ms @ 30fps） │
└──────────┬──────────────────┘
           │
      每 3 幀檢測
           │
    ┌──────▼───────┐
    │ RetinaFace   │ (~100-150ms)
    │ 檢測人臉     │
    └──────┬───────┘
           │
      保存 embedding
           │
    ┌──────▼──────────┐
    │ 顯示結果       │
    │ (100% 幀率)    │
    └────────────────┘

Phase 2b (識別) - 規劃中：
     除上述檢測外，額外：
     - embedding 與 VIP 資料庫比對 (<100ms)
     - 活體檢測（眨眼/搖頭） (<100ms)
     - 決定是否發送 LoRa 通知

設計原則：
- 檢測每 3 幀執行一次，保持 GUI 流暢（全幀率顯示）
- 識別結果快取，跳幀間重複使用
- 優先級：檢測 > 識別 > 表情 > 語音
```

### 📊 幀率控制

- **視覺處理**：10 FPS（足以檢測人臉，降低 CPU 負載）
- **導航速率**：20 Hz（ROS 2 Nav2 標準）
- **HMI 刷新**：10 FPS（觸控屏更新）
- **MQTT 上報**：1 Hz（定期狀態同步）

### 💾 內存優化

- 預分配圖像 buffer，避免重複 malloc
- 只在 GPU 不可用時使用 CPU 推理
- 定期清理 ROS 2 消息隊列（防止內存洩漏）

### 🔄 快速遷移到 Jetson Orin 的架構

為了實現「某人不動，快速移植到 Jetson」的需求，架構設計必須支持：

```yaml
# config/inference_config.yaml
platform: "rpi4"           # 或 "jetson_orin"
model_precision: "int8"    # 或 "fp16" (for Jetson)
batch_size: 1
num_threads: 4
use_gpu: false             # 樹莓派 4 無 GPU
model_paths:
  face_detector: "models/retinaface_int8.onnx"
  face_recognizer: "models/arcface_int8.onnx"

# 若要升級到 Jetson，只需修改 config：
# - 改 platform: "jetson_orin"
# - 改 precision: "fp16"、use_gpu: true
# - 無需修改推理代碼邏輯
```

實現方式：
- 所有推理顯式通過 `InferenceEngine` 抽象層
- 推理引擎根據配置自動選擇後端（ONNX / TFLite / TensorRT）
- 模型路徑、優化參數都在配置文件中

## 5 大核心系統模塊

### 1️⃣ 視覺 AI 辨識模組 (Vision AI)

**文件路徑**: `vision_ai/`
**負責人**: 你（陳佳憲）

#### a. 人臉檢測、活體防偽、身份識別
- **人臉檢測**：RetinaFace / YuNet（檢測率高，適合銀行場景）
- **活體防偽**：
  - 眨眼檢測、搖頭檢測、張嘴檢測（防止照片/視頻欺騙）
  - 關鍵點追蹤（MediaPipe Face Landmarks）+ 運動分析
- **人臉編碼與比對**：ArcFace 輕量版
  - 生成 128D embedding 向量
  - 與 VIP 資料庫進行實時比對
  - 同時檢測黑名單人員

#### b. 表情與情緒分析
- **表情識別**：EfficientNet-B0 微調版（7 種表情）
- **應用**：自動調整迎賓語氣

#### c. 多維度人物特徵分析
- 年齡估計、性別分類等

### 2️⃣ 自主導航與移動模組 (Navigation & SLAM)

**文件路徑**: `robot_navigation/`
**負責人**: 同學（負責機器人運動）

#### a. SLAM 環境掃描與定位
- **感應器**：RPLIDAR A1M8（2D LIDAR）+ RealSense（3D 視覺）
- **實現**：Cartographer / SLAM Toolbox 構建地圖

#### b. 路徑規劃與避障
- **算法**：Nav2 stack (A* + DWA)
- **特性**：即時避開動態障礙物

#### c. 目標跟隨 (Person Following)
- 人臉檢測 → 確定目標 VIP
- LIDAR + 深度相機追蹤目標位置
- PID 控制馬達跟隨目標

### 3️⃣ 語音互動與 HMI 介面 (Interaction)

**文件路徑**: `speech_hmi/`
**負責人**: 你（陳佳憲）

#### a. 語音辨識 (STT)
- **工具**：OpenAI Whisper（邊緣優化版本）
- **功能**：支持中英文，可離線運行
- **識別指令**：「我要開戶」、「我要領錢」等

#### b. 語義理解與指令執行
- 輕量級意圖分類器（fastText / BERT-tiny）
- 決策引擎：VIP → 優先服務，陌生人 → 通用菜單，黑名單 → 告警

#### c. 語音合成 (TTS)
- **工具**：pyttsx3（離線）、Edge-TTS（可選）
- **功能**：個性化迎賓語音、服務指引

#### d. HMI 觸控屏幕 (7 吋觸控屏)
- **UI 應用**：Electron + React 或 Qt
- **功能**：
  - 即時視頻預覽 + VIP 匹配結果
  - 客戶信息面板
  - 銀行信息展示（匯率、號碼牌、理財）
  - 異常告警顯示

### 4️⃣ 遠距通訊與 IoT 通報模組 (Communication)

**文件路徑**: `communication/`
**負責人**: 團隊分工

#### a. LoRa 雙向通訊
- **VIP 通報**：檢測到 VIP → LoRa 推送至行員手持設備
- **安全告警**：檢測到黑名單 / 機器人故障 → 立即警報
- **技術**：LoRa 模組（SX1278） + 行員端 App（Flutter）

#### b. MQTT 與後台連動
- 數據流：VIP 檢測、安全警報、機器人健康狀態、配置同步
- 定期同步本地 SQLite 至後台
- 支持離線緩衝

#### c. 底層硬件通信 (UART)
- 樹莓派 4 與 TurtleBot3 STM32 通信
- 馬達速度指令、編碼器反饋、傳感器數據

### 5️⃣ 系統工作流程 (Workflow Logic)

**文件路徑**: `core_workflow/`

```
[待命] 視覺掃描人臉
   ↓
[檢測] 人臉檢測 + 活體防偽
   ├─ 失敗？→ [告警] 可能欺騙
   └─ 成功
       ↓
[身分判定] 與 VIP/黑名單/訪客比對
   ├─ VIP? → [VIP 流程] 播放個性化語音、顯示貴賓資訊、LoRa 推送
   ├─ 黑名單? → [告警] 立即發送安全警報
   └─ 一般訪客 → [服務流程] 詢問需求、語義理解、提供服務指引
       ↓
[互動導引] 顯示服務信息、語音確認、自主避障導航
   ↓
[日誌記錄] 保存訪客信息、定期 MQTT 同步
```

  - 添加 `recognize_face()` 函數

- [ ] 三重活體檢測

  - **搖頭偵測**：頭部旋轉角度 (yaw) 變化 > 15 度
  - **張嘴偵測**：Mouth Aspect Ratio (MAR) > 0.5
  - **需求**：5 幀內完成 3 種動作才算通過
- [ ] 防欺騙日誌：記錄被偵測為照片/視頻的嘗試

#### 3.3 整合與測試
- [ ] 單元測試：embedding 相似度計算、活體偵測演算法
- [ ] 性能測試：保證檢測(100-150ms) + 識別(<100ms) + 活體檢測(<100ms) = <350ms @ 10 FPS
- [ ] 邊界情況：多人同框、側臉、低光、遮擋、快速動作

## 關鍵設計原則

✅ **邊緣優先、隱私至上**：所有 AI 推理本地執行 (樹莓派)，不上傳人臉圖像（僅 embedding 向量）

✅ **性能約束下的設計**：量化模型、幀率控制、多線程隊列管理

✅ **快速可升級**：配置文件驅動推理引擎，支持一鍵切換到 Jetson Orin

✅ **魯棒容錯**：傳感器故障自動降級、網絡中斷本地緩衝、定期狀態監控

✅ **銀行等級安全**：TLS/SSL 加密、JWT 認證、完整審計日誌

## 風險評估與應對

| 風險 | 概率 | 應對方案 |
|-----|------|--------|
| **樹莓派 CPU 過載** | 中-高 | ① 進一步降幀率（5 FPS）② 申請 Jetson Orin ③ 卸載非關鍵功能 |
| **模型推理延遲超過 500ms** | 中 | 模型進一步蒸餾量化、卸載表情分析 |
| **VNC 連接不穩定** | 低-中 | 改用 WiFi、使用 SSH 進行遠程開發 |
| **LoRa 丟包** | 低 | 使用 MQTT 備份、重傳機制 |
| **內存洩漏** | 低-中 | 定期監控內存使用、優化 ROS 2 消息隊列 |
